# 1项目名称 · 完整总结报告
构建一个集**知识管理**、**技术影响力展示**、**学习轨迹可视化**于一体的个人技术仪表盘。

## 📌 项目概览
- **项目目标**：将散落的 Markdown 笔记、GitHub 活跃度、csdn、掘金社区创作数据聚合在一个可控的私有平台上。
- **项目范围**：导入git hub api、图形可视化
- **关键指标**：
- **当前状态**： 🚧 进行中 
- **形态**：前台展示（面向公众/自己）+ 后台管理（数据/内容维护）。

# 2核心模块
##  2.1知识花园 (Markdown 渲染)

- **功能描述**：将本地或数据库中的 Markdown 文件渲染为美观的网页。
- **技术点**：
    - 支持 `Markdown` 标准语法 + 代码高亮。
    - 支持目录树（TOC）自动生成。
    - **分类管理**：待读、在读、已读/归档，或按技术栈分类（Vue, Node, Rust等）。
    - **检索**：支持全文搜索笔记内容。
## 2.2 数据可视化 (Graphic Dashboard)

- **功能描述**：图形化展示用户的技术活跃度。
- **具体指标**：
    - **GitHub 热力图**：接入 GitHub GraphQL API 或官方 API，展示近一年的贡献日历（绿色方块图）。
    - **平台数据聚合**：展示掘金、思否、知乎、CSDN 等平台的“文章数”、“点赞数”、“粉丝数”、“阅读量”。
    - **学习统计**：展示本周/本月学习时长（如接入 Toggl 或手动记录），或完成的 PR 数量。
## 2.3 后台管理系统 (CMS)

- **功能描述**：提供一个私密入口，用于修改前端展示的数据。
- 
- **功能点**：
    - **笔记管理**：在线编写/修改 Markdown，或通过 Git Webhook 自动同步仓库中的笔记。
    - **数据维护**：手动修改“各平台分数/文章数”（当 API 不可用时作为降级方案，或直接管理数据源）。
    - **配置开关**：切换主题、开关某个模块
## 2.4 智能待办与目标管理

- **描述**：在仪表盘右侧增加一个“Focus”卡片。
- **功能**：
    - **今日目标**：显示当前正在学习的技能（如“刷 LeetCode 3题”）。
    - **里程碑**：展示距离下一个技术目标还剩多少（如“距离 GitHub 1k Star 还差 45”）。
    - **番茄钟**：内置简单的计时器，记录学习时长。
## 2.5 技术栈与能力图谱

- **描述**：替代传统的技能进度条（因为进度条无法衡量技能），改为使用 **“标签云”**或 **“雷达图”**。
    
- **功能**：展示熟练使用的语言、框架，点击标签可快速跳转到对应的笔记分类。
## 2.6 自动化数据同步脚本

- **描述**：为了解决“手动更新各平台数据”的痛点，建议增加一个 **Node 定时脚本**。
    
- **功能**：每天凌晨自动抓取 GitHub、博客园、掘金等平台的公开数据，写入 MySQL，前台展示最新数据。

# 技术栈
前端：vue3、vite、paina、Tailwind CSS、element ui
后端：node、express
数据库：mysql
部署：Nginx

# 3.实施阶段规划

## 第一阶段：基础骨架 (MVP)

1. **环境搭建**：Vue + Express + MySQL 环境配置，Nginx 基础配置。
    
2. **后台登录**：实现简单的 JWT 鉴权。
    
3. **笔记模块**：实现从数据库读取 Markdown，并在前台渲染带目录的文章页。
    
4. **静态展示**：写死几个卡片数据，展示整体 UI 布局。
    

## 第二阶段：数据可视化与集成

1. **GitHub 接入**：后端编写脚本调用 GitHub API，获取用户信息存入数据库，前端生成热力图。
    
2. **平台数据管理**：完成后台 CRUD 接口，支持手动修改“文章数/分数”。
    
3. **动态仪表盘**：前台接入真实 API，替换写死数据。
    
## 第三阶段：增强功能与自动化

1. **定时任务**：编写爬虫脚本或利用官方 API 自动更新各平台数据。
2. **全站搜索**：实现笔记的全文检索。
3. **性能优化**：开启 Gzip，配置 Nginx 缓存策略。

## 4.待确认的技术细节
1. **Markdown 存储方式**：
    直接监听 GitHub 仓库的 Webhook，仓库 Push 时自动同步到数据库（推荐，可实现 Git 管理笔记）。
2. **UI 风格**：
    - 极客风 (深色背景，荧光绿，类似黑客帝国风格)
3. **数据来源**：
    - 如果掘金/知乎等平台没有官方开放 API，你是准备**手动后台填写**，还是编写**模拟登录爬虫**（维护成本高）？建议初期手动填写或仅支持 GitHub 这种有开放 API 的平台。
